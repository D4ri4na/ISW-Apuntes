Apuntes Semana #5

-Git:

	Es un programe en el cual pueden varios progrmadores trabajar al mismo tiempo en un mismo codigo

	Git es un sistema de control de versiones distribuido ,permite trabajar sin conexión o de forma remota con facilidad. 
	Los desarrolladores confirman su trabajo localmente y sincronizan la copia del repositorio con la del servidor. (https://learn.microsoft.com/)

Git branch:

	Nos deja crar nuevas ramas.

	Permite crear una rama nueva (https://www.atlassian.com/)

Git checkout:

	Nos permite cambiar entre las ramas que poseemos

	Te permite desplazarte entre las ramas creadas por git branch . Al extraer una rama, se actualizan los archivos en el directorio de trabajo para reflejar la versión 
	almacenada en esa rama y se indica a Git que registre todas las confirmaciones nuevas en dicha rama. (https://www.atlassian.com/)

Git remote:

	Comando que permite a todos los integrantes intercambiar sus cambios.

	Permite crear, ver y eliminar conexiones a otros repositorios. Las conexiones remotas se parecen más a marcadores que a enlaces directos a otros repositorios.(https://www.atlassian.com/)

Git clone:

	Clona un repositorio en un nuevo repositorio

	Apunta a un repositorio existente y crear un clon o una copia del repositorio de destino.(https://www.atlassian.com/)

Git merge:

	Une ramas independientes en una sola rama principal.

	Une dos o más historiales de desarrollo(https://git-scm.com/)

Git fork:

	Copia de un repositorio existente en la cuenta del usuario.

	Se encarga de la creación de una copia de un repositorio en la cuenta de usuario.(https://keepcoding.io/)
